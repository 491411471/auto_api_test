# 补订单接口重试机制优化说明

## 问题描述

当提交补订单接口时，如果返回以下错误：
```json
{
  "errorMessage": "订单中已存在闲鱼指定商家，请勿重复补订单"
}
```

之前的逻辑会用**同一个订单号**尝试不同的用途，但这样仍然会失败，因为订单本身已经有了该用途类型。

## 解决方案

### 核心改进

**当遇到"已存在"类错误时，自动查询新的订单号并重试，而不是用同一个订单号尝试不同用途。**

### 修改的文件

#### 1. `data/scenario/order/repair_order.yaml`

**修改前：**
```yaml
retry_config:
  max_attempts: 3
  interval_seconds: 5
  error_keyword: "订单中已存{name}，请勿重复补订单"
```

**修改后：**
```yaml
retry_config:
  max_attempts: 3
  interval_seconds: 5
  # 支持多种错误关键词（满足任一即触发重试）
  error_keywords:
    - "订单中已存在{name}，请勿重复补订单"
    - "订单中已存在闲鱼指定商家，请勿重复补订单"
```

#### 2. `testcases/scenario/order/test_repair_order.py`

**主要改进：**

1. **支持多个错误关键词**
   - 兼容旧的 `error_keyword` 格式
   - 支持新的 `error_keywords` 列表格式
   - 自动匹配任一错误关键词

2. **记录已尝试的订单号**
   ```python
   tried_order_ids = set()  # 新增：记录已尝试过的订单号
   ```

3. **重试时查询新订单**
   ```python
   # 如果是重试且已尝试过订单，修改SQL查询排除已尝试的订单
   if tried_order_ids:
       order_ids_str = ','.join([f"'{oid}'" for oid in tried_order_ids])
       sql = sql_template.replace('${shop_id}', shop_id)
       # 在 LIMIT 1 前添加 NOT IN 条件
       if 'LIMIT' in sql:
           sql = sql.replace('LIMIT', f"AND order_id NOT IN ({order_ids_str}) LIMIT")
   ```

4. **智能错误匹配**
   ```python
   # 遍历所有错误关键词，检查是否匹配
   for keyword_template in error_keywords:
       if '{name}' in keyword_template:
           dynamic_keyword = keyword_template.format(name=repair_name)
       else:
           dynamic_keyword = keyword_template
       
       if dynamic_keyword in error_msg:
           is_retryable_error = True
           break
   ```

## 工作流程

### 第一次尝试
1. 查询订单A
2. 获取用途列表，选择用途X
3. 提交补订单（订单A + 用途X）
4. ❌ 返回错误："订单中已存在闲鱼指定商家"

### 第二次尝试（自动重试）
1. ✅ **查询新订单B**（排除订单A）
2. 获取用途列表，选择用途Y
3. 提交补订单（订单B + 用途Y）
4. ✅ 成功！

### 第三次尝试（如果第二次也失败）
1. ✅ **查询新订单C**（排除订单A和B）
2. 获取用途列表，选择用途Z
3. 提交补订单（订单C + 用途Z）
4. ✅ 成功或失败（达到最大重试次数）

## 配置说明

### 添加新的错误关键词

如果以后还有其他"已存在"类错误，只需在 `repair_order.yaml` 中添加：

```yaml
retry_config:
  error_keywords:
    - "订单中已存在{name}，请勿重复补订单"
    - "订单中已存在闲鱼指定商家，请勿重复补订单"
    - "订单中已存在NEW_ERROR_TYPE，请勿重复补订单"  # 新增
```

### 调整重试参数

```yaml
retry_config:
  max_attempts: 5          # 最大重试次数（默认3）
  interval_seconds: 10     # 重试间隔秒数（默认5）
```

## 优势

1. ✅ **避免无效重试**：不再用同一个订单号尝试不同用途
2. ✅ **智能订单选择**：自动排除已尝试的订单，查询新订单
3. ✅ **灵活错误匹配**：支持多种错误关键词，易于扩展
4. ✅ **向后兼容**：兼容旧的配置格式
5. ✅ **详细日志记录**：清晰记录每次重试的过程

## 注意事项

1. 确保数据库中有足够的可补订单，否则重试可能因无订单可用而失败
2. 每次重试都会等待 `interval_seconds` 秒，总耗时 = 重试次数 × 间隔时间
3. 如果所有可用订单都尝试过仍失败，测试会终止并报告错误
